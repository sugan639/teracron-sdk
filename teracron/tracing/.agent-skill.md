# Teracron Tracing — Agent Skill Reference

> Complete AI agent reference for `@trace`, event querying, CLI auth,
> failure diagnosis, curl commands, and workflow inspection.
> SDK version: `0.6.0+`

---

## Overview

The Teracron SDK provides:

| Capability | Use Case |
|---|---|
| `@trace("workflow")` | Decorate functions — automatic span lifecycle |
| `trace_context("workflow")` | Context manager — trace arbitrary code blocks |
| `get_trace_header()` / `set_trace_header()` | Cross-process propagation via HTTP headers |
| `trace_sample_rate` | Deterministic hash-based sampling (0.0–1.0) |
| `tracing_scrubber` | User-provided PII scrubber callable |
| `teracron-agent login` | CLI authentication (like Convex) |
| `teracron-agent events` | Query workflow events for diagnosis |
| `teracron-agent simulate` | Replay failed traces locally |
| **curl + API key** | Direct HTTP access to Teracron query API |

## 1. CLI Authentication

### Login (store credentials)

```bash
# Interactive prompt:
teracron-agent login

# With API key directly:
teracron-agent login --api-key tcn_...

# Or via positional arg:
teracron-agent login tcn_...

# With custom domain:
teracron-agent login --api-key tcn_... --domain api.teracron.com
```

Credentials are stored at `~/.teracron/credentials.json` (mode `0600`).

### Check auth status

```bash
teracron-agent whoami
teracron-agent whoami --json
```

### Logout

```bash
teracron-agent logout
```

### Auth resolution priority

1. `--api-key` CLI flag (highest)
2. `TERACRON_API_KEY` environment variable
3. `~/.teracron/credentials.json` (lowest)

## 2. `@trace` Decorator

### Basic (no parameter capture)

```python
from teracron import trace

@trace("payment")
def create_order(cart):
    order = Order.create(cart)
    return order
```

### With Parameter Capture (opt-in)

```python
@trace("payment", capture=["order_id", "amount"])
def charge_card(order_id, amount, card_number):
    # ✅ order_id and amount are captured
    # ❌ card_number is NOT captured
    gateway.charge(order_id, amount, card_number)
```

### Nested Spans

```python
@trace("payment")
def process_payment(order_id, amount, card_number):
    validate_order(order_id)           # child span
    charge_card(order_id, amount, card_number)  # child span
```

## 3. Querying Events (AI Agent Use-Case)

### Via CLI

```bash
# List recent failed events
teracron-agent events --status=failed --json

# Filter by workflow
teracron-agent events --workflow=payment --limit=10 --json

# Get a full trace span tree
teracron-agent trace <TRACE_ID> --json

# List workflow summaries
teracron-agent workflows --json
```

### Via curl (Direct API Access)

```bash
# Set your API key
export TERACRON_API_KEY="tcn_..."

# List recent failed events
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/events?status=failed&limit=10"

# Events for a specific workflow
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/events?workflow=payment&limit=20"

# Full trace span tree
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/traces/<TRACE_ID>"

# Workflow summaries
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/workflows?limit=20"

# Single span detail
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/spans/<SPAN_ID>"

# Auth check
curl -s -H "Authorization: Bearer $TERACRON_API_KEY" \
  "https://www.teracron.com/v1/auth/whoami"
```

## 4. Diagnosing Failures (AI Agent Decision Tree)

When a user reports an issue, follow this sequence:

### Step 1: Query recent failed events

```bash
teracron-agent events --workflow=<workflow_name> --status=failed --json
```

### Step 2: Identify the trace_id

Parse the JSON output. Pick the most recent `trace_id` with `status: "failed"`.

### Step 3: Fetch full span tree

```bash
teracron-agent trace <trace_id> --json
```

### Step 4: Analyze the span tree

- Look at the `status` field of each span.
- Find spans with `status: "failed"`.
- Check `error_type` and `error_message` for root cause.
- Check `captured_params` for input values that caused the failure.

### Step 5: Generate a simulation (optional)

```bash
# Get markdown diagnosis
teracron-agent simulate <trace_id> --format=markdown

# Get JSON context
teracron-agent simulate <trace_id> --format=json

# Generate a Python repro script
teracron-agent simulate <trace_id> --format=script > repro.py
```

### Step 6: Run the repro script locally

```bash
python repro.py
```

### Step 7: Apply fix and verify

After fixing the issue, verify by re-running the simulation script.

## 5. Error Handling

| HTTP Status | Meaning | Agent Action |
|---|---|---|
| `200` | Success | Parse JSON response |
| `401` | Invalid API key | Run `teracron-agent login` |
| `404` | Endpoint not deployed | Backend not updated yet — wait or check docs |
| `429` | Rate limited | Wait `Retry-After` seconds, then retry |
| `0` | Connection failed | Check network, verify domain is reachable |

### When backend returns 404

The SDK query client and CLI are ready for the Teracron query API, but the
backend endpoints (`/v1/events`, `/v1/workflows`, `/v1/traces/:id`) may not
be deployed yet. The SDK returns:

```json
{
  "error": "Endpoint not found — this API is not yet deployed on the backend.",
  "hint": "The SDK is ready for this feature, but the Teracron backend has not yet been updated.",
  "status_code": 404
}
```

## 6. Sampling

```python
teracron.up(trace_sample_rate=0.1)  # 10% of traces
```

- Decision made once at the trace root (deterministic MD5 hash of `trace_id`).
- All spans in a sampled trace are kept (all-or-nothing).
- Same `trace_id` → same decision across services.
- `rate=1.0` = capture everything (default).
- `rate=0.0` = capture nothing.
- Functions still execute and exceptions propagate even when not sampled.

## 7. PII Scrubber

```python
def my_scrubber(data: dict) -> dict:
    data.pop("email", None)
    data.pop("ssn", None)
    return data

teracron.up(tracing_scrubber=my_scrubber)
```

- Applied to `metadata` and `captured_params` before buffering.
- Receives a **shallow copy** — original data is never mutated.
- If scrubber raises → data is **dropped** (never leaked).
- If scrubber returns non-dict → data is dropped.
- If scrubber returns empty dict → data becomes `None`.

## 8. Structured Events (Auto-Emission)

Enable workflow lifecycle events:

```python
teracron.up(trace_emit_events=True)
```

When enabled, the `@trace` decorator auto-emits:
- `workflow_started` / `workflow_completed` / `workflow_failed` (root spans)
- `step_started` / `step_completed` / `step_failed` (child spans)

Events are separate from spans — they represent discrete moments, not durations.
They power the Teracron workflow timeline page.

## 9. Framework Middleware

```python
from teracron.tracing.middleware.fastapi import TeracronTracingMiddleware
app.add_middleware(TeracronTracingMiddleware, workflow="api")
```

## 10. Configuration

| Parameter | Env Variable | Default | Range |
|---|---|---|---|
| `tracing_enabled` | `TERACRON_TRACING_ENABLED` | `True` | bool |
| `trace_batch_size` | `TERACRON_TRACE_BATCH_SIZE` | `100` | 1–10,000 |
| `trace_flush_interval` | `TERACRON_TRACE_FLUSH_INTERVAL` | `10.0` | 1.0–300.0 |
| `trace_sample_rate` | `TERACRON_TRACE_SAMPLE_RATE` | `1.0` | 0.0–1.0 |
| `trace_emit_events` | `TERACRON_TRACE_EMIT_EVENTS` | `False` | bool |
| `tracing_scrubber` | — | `None` | callable |

